## Current-thread provenance boundary

The 2026-07-14 through 2026-07-15 Codex work refocused the product from Agenty to Agents SDK, built and verified the local v0.1 surface, renamed the repository in place, and produced the dated handoff package. Public claims are supported only by checked-in source, tests, explicit verification records, Git history, and the user-authored prompts preserved in `prompts.md`.

The handoff does not claim access to an export of hidden model reasoning or a complete Codex platform event log. It records observable actions, commands, files, decisions, results, and user-visible thread context. Secrets, credential values, private memory, paid assets, and unreviewed client material are excluded.

The guard was not bypassed, no task file was mutated, and the existing 13-task `project-context/tasks.json` remains the canonical ledger. This local project-descriptor issue is distinct from the remote credential’s missing `write:brain` scope; both must be resolved through supported You.md authentication/registration flows rather than hand edits.

## Promotion record format

Every promoted primitive records source path/revision, generalized behavior, excluded private details, new owner, license, tests, documentation, reviewer, and replacement/deprecation relationship. Until that record and acceptance evidence exist, the item is a proposal, not an Agents SDK registry item.
