# Vision

Agents SDK is **the open-source component system for building production agentic applications**.

It helps developers build the entire application around an agent, not only the model call or loop. The product is a curated ecosystem of source-owned components, agents, skills, tools, MCP integrations, patterns, templates, examples, CLI workflows, and documentation.

## Product promise

A developer can discover a proven building block, understand when and why to use it, install its source, run the example and tests, customize it without fighting an abstraction, and retain ownership of the code.

## Product areas

1. Components
2. Agents & Skills
3. Tools & MCP
4. Patterns
5. Templates
6. Examples
7. CLI
8. Docs

These are durable information-architecture categories, not claims that every category is already populated.

## What Agents SDK is not

- Not an agency landing page or lead-generation wrapper.
- Not a thin model or provider SDK.
- Not a universal provider-neutral runtime.
- Not a hosted component marketplace.
- Not a private prompt dump or automatic cross-project code copier.
- Not a substitute for application-owned auth, data, tenancy, permissions, and deployment policy.

## End state

Agents SDK becomes the place developers reach for after a basic agent demo stops being enough. Its moat is the quality of the source, explanations, examples, production lessons, and installation experience, not the number of category pages.

BAMF may offer custom development and enterprise support, but the open-source developer ecosystem remains the primary product and primary homepage narrative.
